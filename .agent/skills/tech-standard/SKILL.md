# 💻 Tech Standard (技術・開発運用標準)

## 概要
AIエージェントがコードを記述、管理、調査する際の技術・運用の標準化を行います。

## 技術標準

### 1. Git同期とバージョン管理
- **コミット**: 意味のある単位（Atomic Commit）で作業を区切る。
- **同期**: セッション終了時は必ず `git push` を提案・実行する。
- **🚨 セキュリティ制限**: 職場プロジェクトの場合、認可されていない個人リポジトリ等へのPushは厳禁。不明な場合は必ずユーザーに確認すること。

### 2. Mermaid図解 (HD)
- 複雑なロジックやデータフローは原則としてMermaidで図解する。
- **引用符ルール**: ノード内のテキストは必ず二重引用符 `""` で囲む。
- 詳細は `references/mermaid_guide.md` を参照。

### 3. Claude Code / LLM の使役
- **起動**: `launcher.ps1` 等の正規スクリプトを経由して適切なLLMプロキシを選択する。
- **CLAUDE.md**: プロジェクト固有の規約がある場合は `CLAUDE.md` を最優先で遵守する。

### 4. ハック & リサーチ手法
新しい技術の導入時は、以下の3点を徹底する。
1. **可視化**: Mermaidでアーキテクチャを図解。
2. **比較**: 代替案との比較（Why）を記録。
3. **計画**: 実装前にプラン（`.agent/plans/`）を作成（`design-docs` Skill を併用）。
