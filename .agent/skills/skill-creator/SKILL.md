# 🛠 Skill Creator (スキル作成ガイド)

このスキルは、効果的なスキルを作成するための指針を提供します。

## 💡 スキルについて

スキルとは、専門知識、ワークフロー、およびツールを提供することで、Claude の機能を拡張するモジュール式のパッケージです。特定のドメインやタスクのための「オンボーディング・ガイド」と考えてください。これにより、汎用的なエージェントから、モデル単体では持ち得ない手順知識を備えた専門的なエージェントへと Claude を変貌させます。

### スキルが提供するもの

1.  **専門的なワークフロー**: 特定のドメインにおける多段階の手順。
2.  **ツール統合**: 特定のファイル形式や API を操作するための指示。
3.  **ドメインの専門知識**: 企業固有の知識、スキーマ、ビジネスロジック。
4.  **リソースのセット**: 複雑で反復的なタスクのためのスクリプト、参照資料、アセット。

## 🎯 コア・プリンシパル (基本原則)

### 「簡潔さ」が命 (Concise is Key)

コンテキスト・ウィンドウは貴重な資源です。スキルは、システムプロンプト、会話履歴、他のスキルのメタデータ、そして実際のユーザーリクエストとウィンドウを共有します。

**デフォルトの前提: Claude はすでに十分に賢い。** Claude がまだ持っていないコンテキストのみを追加してください。すべての情報に対して、「Claude は本当にこの説明を必要としているか？」「この段落はトークンコストに見合う価値があるか？」と問い直してください。冗長な説明よりも、簡潔な例示を優先してください。

### 適切な自由度の設定

タスクの繊細さと多様性に合わせて、具体性のレベルを調整します：

-   **高い自由度 (テキストベースの指示)**: 複数のアプローチが有効な場合、状況に応じて判断が必要な場合、またはヒューリスティックな指針で済む場合に使用します。
-   **中程度の自由度 (パラメータ付きの疑似コードやスクリプト)**: 推奨されるパターンが存在し、ある程度のバリエーションが許容される場合に使用します。
-   **低い自由度 (特定のスクリプト、少ないパラメータ)**: 操作が繊細でエラーが発生しやすい場合、一貫性が極めて重要な場合、または特定の順序を厳守する必要がある場合に使用します。

### スキルの構成 (Anatomy)

すべてのスキルは、必須の `SKILL.md` ファイルと、オプションのリソースで構成されます：

```text
skill-name/
├── SKILL.md (必須)
│   ├── YAML フロントメター (必須)
│   │   ├── name: (必須)
│   │   └── description: (必須)
│   └── Markdown 指示書 (必須)
└── リソース (任意)
    ├── scripts/    - 実行可能なコード (Python/Bash など)
    ├── references/ - 必要に応じてコンテキストにロードされるドキュメント
    └── assets/     - 出力で使用されるファイル (テンプレート、アイコン、フォントなど)
```

#### SKILL.md (必須)
-   **フロントメター (YAML)**: `name` と `description` を含みます。Claude はこれだけを読んでスキルの使用可否を判断するため、スキルの内容と使用場面を明確かつ包括的に記述することが非常に重要です。
-   **ボディ (Markdown)**: スキルを使用するための指示とガイド。スキルがトリガーされた後にのみ読み込まれます。

#### リソース (任意)
-   **スクリプト (`scripts/`)**: 決定論的な信頼性が必要なタスクや、繰り返し書かれるコードのための実行可能ファイル。
-   **参照資料 (`references/`)**: Claude の思考やプロセスを補助するために、必要に応じてロードされるドキュメント。API ドキュメントやスキーマなど。
-   **アセット (`assets/`)**: コンテキストにはロードせず、Claude が生成する出力物（テンプレートやロゴなど）として使用されるファイル。

## ⏩ 段階的開示 (Progressive Disclosure)

コンテキストを効率的に管理するため、3段階のロード・システムを使用します：

1.  **メタデータ (name + description)**: 常にコンテキストに存在 (~100 語)
2.  **SKILL.md ボディ**: スキルがトリガーされたときにロード (<5,000 語)
3.  **リソース**: Claude が必要と判断したときにのみロード (実行のみならトークン消費なし)

## 🏗 スキル作成プロセス

1.  **具体例による理解**: スキルがどのように使われるかを具体的に把握する。
2.  **再利用可能な中身の計画**: スクリプト、参照資料、アセットをリストアップする。
3.  **初期化**: `init_skill.py` を実行してディレクトリを作成する。
4.  **編集**: リソースを実装し、`SKILL.md` を執筆する。
5.  **パッケージング**: `package_skill.py` を実行して配布用ファイルを作成する。
6.  **反復 (イテレーション)**: 実際の使用結果に基づいて改善する。

### ステップ 3: スキルの初期化 (詳細)
新しいスキルをゼロから作成する場合は、常に `init_skill.py` を使用してください。

```bash
python scripts/init_skill.py <skill-name> --path <output-directory>
```

### ステップ 5: スキルのパッケージング (詳細)
開発が完了したら、配布可能な `.skill` ファイルにパッケージ化します。この際に自動バリデーションが実行されます。

```bash
python scripts/package_skill.py <path/to/skill-folder>
```

## 📋 執筆ガイドライン

-   **フロントメターの `description`**: これがスキルのメイン・トリガーです。「何をするか」と「いつ使うか」の両方を含めてください。
-   **命令形/不定詞を使用**: 指示は明確に、動作を促す形で記述してください。
-   **不要なファイルを含めない**: `README.md` や `INSTALLATION_GUIDE.md` などの、AI エージェントが作業する上で不要な説明ファイルは作成しないでください。
